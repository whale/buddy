Overview

One system, three pressure levels

Buddy re-themes itself by active task count, entirely through CSS custom properties. Every surface reads from tokens so it adapts automatically. The cardinal rule: never hardcode an adaptive color — bind it to a token. This guide is the visual mirror of that system and flags where the code drifts from it.

lvl0 — normal
≤4 active · black on white
lvl1 — warning
5 active · text turns red
lvl2 — full
6 active · whole drawer red, text light

Task rows — neutral · focused · done

Active task text is the deliberate exception to the token rule: it uses escalation color directly (black → red → white). The "now"/focused row gets a subtle grey (a darkening overlay on red).

Done label rotation

The completed-row label rotates through 25 calm phrases, picked deterministically per task id (stable, equal-weight). Title Case, each ends in "!".

Done tab — completed & skipped

Completed past tasks show the done word + strikethrough. Skipped (undone) tasks show plain, with a restore arrow (↩) to send them back to today.

Audit

Discrepancies to tighten

Verified against dist/index.html. The system is largely consistent (adaptation via class hooks + lvl2 overrides). Below: what was just resolved, and the remaining consistency opportunities — none currently break legibility on red.

StatusItemRecommendation
FixedDone-tab day header was text-black (illegible on lvl2 red)Now var(--ink). (PR #46)
FixedSkip base color set inline → defeated its :hoverMoved base color into CSS so hover applies. (PR #45)
FixedBuddy! hover lightened on red (against the darken idiom)Darkens on lvl1/lvl2; lightens only the black lvl0 pill. (PR #45)
MedIcon stroke width inconsistent: 2.2 (+, ↩, ✕) · 2.0 (edit, sleep, calendar) · 1.4 (weather)Pick a deliberate scale (e.g. 1.75 default, 1.4 large decorative) and apply uniformly.
LowRedundant borders: rows carry both .bd and literal border-[#d9d9d9]The .bd + .lvl2 .bd override already handles adaptation; drop the literal.
LowHover surfaces vary: row actions use raw hover:bg-black/10; chrome uses --chrome-hoverRoute all hover tints through one token (--chrome-hover).
LowSettings adapts via [class*="text-black"] / [class*="border-[#d9d9d9]"] substring overridesFragile. Prefer the same --ink / .bd hooks the rest of the app uses.
Low.lvl2 .chrome{color:#fff !important} flattens opacity (e.g. text-black/40 → full white)Define a dim chrome token so secondary glyphs stay secondary on red.
LowDone label is 15px in the today list but 18px in the Done tabConfirm intent; align if it should read as one component.
